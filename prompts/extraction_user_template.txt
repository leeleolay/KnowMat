Follow the laboratory extraction semantics as the source of truth.

Step 1: Global metadata and role distinction

- Extract the paper title and DOI into the root `Paper_Metadata` object.
- Extract this paper's own materials as `Target`.
- If explicit cited benchmark values are present, you may extract them as separate `Reference` items, but never mix them into a `Target` item.

Step 2: Item decomposition

- One item = one unique combination of composition + process.
- Split whenever composition, process state, sample/state label, route, orientation/loading mode, parameter level, or graded step changes.
- If the same item has multiple testing conditions, keep one item and place all tests in that item's `Properties_Info`.
- If a variant is explicitly named but only sparse evidence exists, still keep a sparse item.

Step 3: Composition extraction (`Composition_Info`)

- Fill `Role`, `Alloy_Name_Raw`, `Nominal_Composition`, and `Measured_Composition`.
- Keep nominal and measured composition strictly separate.
- Normalize reported composition to 100.0, using `other` only when justified by residual balance.
- Preserve mature commercial alloy names with null normalized arrays if no explicit numeric composition is reported.
- Never convert trade names or polymer abbreviations into fake element formulas.

Step 4: Process extraction (`Process_Info`)

- Build `Process_Text.original` and `Process_Text.simplified`.
- Extract quantified variables into `Key_Params`.
- Use canonical parameter names where possible, and dynamic keys with unit hints when needed.
- For multistage processing, use stage suffixes such as `_Stage1_K` and never average stages.

Step 5: Microstructure extraction (`Microstructure_Info`)

- Build `Microstructure_Text.original` and `Microstructure_Text.simplified`.
- Extract `Main_Phase`, `Porosity_pct`, `Relative_Density_pct`, `Grain_Size_avg_um`, `Precipitate_Size_avg_nm`, and `Precipitate_Volume_Fraction_pct` whenever evidence exists.
- Put advanced quantified microstructure metrics under `Advanced_Quantitative_Features`.

Step 6: Properties extraction (`Properties_Info`)

- Extract all reported properties for the same item.
- Keep strict ordering from Tier 1 to Tier 3.
- Use compressive property names only for compressive contexts.
- `Test_Temperature_K` is only the actual testing environment temperature, not a process temperature.
- For ranges, keep midpoint in `Value_Numeric` and source range in `Value_Range`.
- For mean±std, keep mean in `Value_Numeric` and std in `Value_StdDev`.

Step 7: Table alignment and generalization checks

- Reconstruct by header-lock, row-lock, temperature-lock, and unit-lock.
- Do not guess from ambiguous OCR or unlabeled plot lines.
- Build the full list of explicit labels and parameter levels before finalizing output.
- Ensure every clearly supported variant has a corresponding item.
- Do not overfit to one phrase pattern; preserve generalized extraction behavior across different paper layouts.

Step 8: Mandatory final self-audit

- Check Target/Reference purity.
- Check no explicit variant, test temperature, or property column is silently dropped.
- Check MPa/GPa values are not assigned to percentage properties, and percentage values are not assigned to strength fields.
- Check the final payload is valid JSON only.

Important output rule:

- Follow the laboratory schema semantics rooted at `Paper_Metadata` and `items`, but emit the runtime internal schema rooted at `compositions`.
- Use field mapping consistent with the system prompt so downstream pipeline stages remain compatible.
- Do not output any prose or markdown.
